Цель работы

Целью проекта является создание приложения, предоставляющее набор инструментом для работы над множеством задач планирования.

Функциональные требования

  1. Реализация набора команд для работы с задачами
    1. create - команда для создания задач
    2. update - команда для обновления задач
    3. delete - команда для удаления задач
  2. Реализация набора технических команд для расширения функционала и облегчения работы над задачами
    1. root - сменить корневую задачу
    2. alias - позволяет менять набор кастомных команд, создающихся пользователем на основе существующих команд
    3. copy - создаёт новую задачу на основе уже существующих
    4. delay - сдвигает временные рамки задач на заданную величину
    5. actualize - подгоняет повторяющиеся задачи под текующее время
    6. help - выводит информацию о командах
  3. Реализация набора команд, позволяющих различными способами визуально представлять описаное пользователем множество задач
    1. grid - вывод задач в виде таблицы/решётки
    2. calendar - вывод задач на календаре
    3. tree - вывод задач в виде дерева

Все перечисленные выше команды реализуются как через интерфейс командной строки, так и через визуальный интерфейс.

Описание системы

Всего в системе существует 7 сущностей:

  1. root элемент - корневой элемент, определяет отношение задач к какой-то одной теме. Может иметь пароль для того, чтобы ограничивать доступ к получению задач.
  2. базовый элемент - является "указателем" на задачу, содержит базовую информацию и идентификатор, по которому можно найти остальные элементы, которые с ней связаны
  3. элемент чеклиста - содержит чеклист, связанный с задачей
  4. элемент журнала - содержит журнал (набор текстовых страниц), связанных с задачей
  5. элемент связи - содержит информацию о связи задачи с другими задачами из группы и её характере
  6. элемент времени - содержит информацию о временных ограничениях для данной задачи

Каждая сущность описывает конкретный элемент данных, связанный с задачей.

У каждого элемента, не являющегося root или базовым элементом есть поля rootid и bid, соответственно определяющие связь элемента с root и базовым элементом задачи.
